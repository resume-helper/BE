## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **서비스명** | Atomic CV |
| **분류** | AI 기반 이력서 관리·피드백 플랫폼 |
| **핵심 컨셉** | 콘텐츠 블록을 한 번 쌓아두면 PDF·웹 이력서를 동시에 뽑아내는 **원소스 멀티아웃풋** 구조 |
| **플랫폼** | Next.js 기반 반응형 웹 (Mobile / Desktop 동시 대응) |
| **타겟 유저** | 취업 준비생, 이직 준비 중인 직장인 |
| **1차 MVP 목표** | 가입자 1,000명 / PDF 생성률 90% / 피드백 수신율 40% |

### 서비스 한 줄 정의

> "경력·기술·프로젝트 블록을 한 번 작성하면, 지원처마다 블록을 조합해 PDF와 웹 이력서를 즉시 생성하고, 채용담당자의 실시간 피드백까지 받을 수 있는 이력서 플랫폼"

## 2. 사용자 문제 & 해결 방향

| # | 사용자 Pain Point | Atomic CV 해결 방법 |
|---|------------------|-------------------|
| 1 | 지원처마다 이력서를 반복 작성해야 한다 | 콘텐츠 블록 라이브러리 — 한 번 작성 후 블록 조합으로 재사용 |
| 2 | 피드백 채널이 없다 (AI·지인에만 의존) | 채용담당자용 피드백 링크 제공 — 별점·코멘트·태그 수집 |
| 3 | 이력서를 채용담당자가 읽는지 알 수 없다 | 웹 이력서 열람 분석 — 체류시간·섹션별 히트맵 제공 |
| 4 | 성과를 정량화하고 구조화하기 어렵다 | Claude AI 이력서 클리닉 — *(Phase 2, MVP 이후)* |

## 4. 기능 상세 명세

### F-01. 로그인

**목적**: 서비스 진입점, 회원 식별

**지원 방식**:
- 소셜 로그인: Google OAuth2, Kakao OAuth2, Naver OAuth2
- ~~일반 로그인 (이메일 + 비밀번호)~~ — 미사용 (2026-05-02 결정)

**인증 구조**:
- Access Token: JWT (단기 만료, HttpOnly Cookie 전달 — XSS 방어)
- Refresh Token: Redis 저장 (장기 만료)
- 로그아웃: Redis Blacklist로 토큰 무효화

~~**이메일 인증**~~ — 미사용 (소셜 로그인 전용으로 결정, 2026-05-02)

**수용 기준 (Acceptance Criteria)**:
- [x] 소셜 로그인 3종(Google, Kakao, Naver) 모두 동작
- [x] 로그인 후 JWT Access Token / Refresh Token 발급
- [x] Refresh Token으로 Access Token 재발급
- [x] 로그아웃 시 토큰 즉시 무효화

---

### F-02. 로그아웃

**목적**: 세션 종료, 토큰 무효화

**수용 기준**:
- [ ] 로그아웃 API 호출 시 Refresh Token Redis에서 삭제
- [ ] Access Token Redis Blacklist에 등록
- [ ] 이후 해당 Access Token으로 API 호출 시 401 응답

---

### F-03. 마이페이지

**목적**: 프로필 조회 및 수정

**제공 정보**: 이름, 이메일, 프로필 이미지, 연동 소셜 계정 목록

**수용 기준**:
- [ ] 프로필 정보 조회 API
- [ ] 이름 / 프로필 이미지 수정 API
- [ ] 연동된 소셜 계정 목록 조회

---

### F-04. 콘텐츠 블록 라이브러리

**목적**: 이력서 구성 콘텐츠를 블록 단위로 관리

**블록 타입**: 경력(CAREER), 기술스택(SKILL), 프로젝트(PROJECT), 학력(EDUCATION), 자격증(CERTIFICATE), 활동(ACTIVITY), 커스텀(CUSTOM)

**핵심 동작**:
- 블록 생성 / 조회 / 수정 / 삭제 (Soft Delete)
- 자동 저장 (Draft 상태 동기화)
- 블록 순서 변경 (order_index)

**수용 기준**:
- [ ] 블록 타입별 CRUD API
- [ ] Draft 자동 저장 API (일정 주기로 FE가 호출)
- [ ] 블록 순서 일괄 변경 API

---

### F-05. PDF 빌더

**목적**: 블록을 조합하여 PDF 이력서 생성

**역할 분담**:
- **FE**: 블록 데이터를 받아 PDF 렌더링 및 생성 (클라이언트 사이드)
- **BE**: 블록 조합 데이터 API 제공 + FE가 생성한 PDF 파일 S3 업로드 API 제공

**ATS 준수사항** *(FE 구현 시 필수)*:
- 텍스트 레이어 유지 (이미지 PDF 불가)
- 폰트 임베딩 필수

**수용 기준**:
- [ ] 선택 블록 조합 데이터 반환 API
- [ ] PDF 파일 S3 업로드 및 다운로드 URL 발급 API
- [ ] 이미지 업로드 API (용량 5MB 이하, jpg·png·webp)

---

### F-06. 회원탈퇴

**목적**: 계정 및 관련 데이터 삭제

**수용 기준**:
- [ ] 탈퇴 요청 시 users.deleted_at 설정 (Soft Delete)
- [ ] 탈퇴 후 로그인 불가
- [ ] 데이터 보존 정책 고지 (30일 후 완전 삭제 예정)

---

### F-07. 웹 이력서 발행

**목적**: PDF 없이 링크로 공유 가능한 웹 이력서 제공

**핵심 동작**:
- 발행 시 UUID 슬러그 생성 (`/r/{slug}`)
- 버전 스냅샷 생성 (발행 시점 블록 데이터 JSON 저장)
- 발행 / 비공개 토글

**수용 기준**:
- [ ] 웹 이력서 발행 API (슬러그 생성)
- [ ] 버전 스냅샷 저장 (resume_versions)
- [ ] 슬러그로 이력서 공개 조회 API (비인증 접근 가능)
- [ ] 비공개 처리 시 404 응답

---

### F-08. 유저 대시보드

**목적**: 작성한 이력서 및 버전 목록 관리

**제공 정보**: 이력서 목록, 버전 히스토리, 발행 상태, 최근 열람 수

**수용 기준**:
- [ ] 유저의 이력서 목록 조회 API
- [ ] 이력서별 버전 목록 조회 API
- [ ] 버전 롤백 API

---

### F-09. 웹 이력서 열람 분석

**목적**: 채용담당자가 이력서를 얼마나 읽었는지 파악

**수집 데이터**:
- 방문자 IP, User-Agent, Referrer
- 전체 체류시간
- 섹션(블록)별 체류시간

**집계 데이터**: 섹션별 히트맵, 평균 체류시간, 방문 횟수

**수용 기준**:
- [ ] 열람 세션 시작/종료 API
- [ ] 섹션별 체류시간 수집 API
- [ ] 집계 데이터 조회 API (히트맵용)
- [ ] 동일 IP 중복 카운트 방지

---

### F-10. 채용담당자 피드백

**목적**: 이력서를 본 채용담당자에게 익명 피드백 수집

**수집 항목**: 별점(1~5), 텍스트 코멘트, 태그(강점/약점 등)

**어뷰징 방지**: 동일 IP Rate Limit, 비속어 필터

**수용 기준**:
- [ ] 피드백 제출 API (비인증 접근 가능)
- [ ] 피드백 태그 다중 첨부
- [ ] 동일 IP Rate Limit (1시간 내 3회 제한)
- [ ] 피드백 수신 시 이력서 소유자에게 알림

## 5. 비기능 요구사항

| 항목 | 요구사항 |
|------|---------|
| **응답 시간** | 일반 API 평균 200ms 이하, PDF 업로드 제외 |
| **가용성** | MVP 단계 SLA 미정, EC2 단일 인스턴스 운영 |
| **보안** | JWT + Redis Blacklist, HTTPS 필수, SQL Injection 방어 |
| **확장성** | Modular Monolith → MSA 전환 가능 구조 유지 |
| **파일 용량** | 이미지 5MB 이하, PDF 20MB 이하 |
| **동시 접속** | MVP 목표 동시 접속자 100명 이하 |

## 6. 제외 범위 (Out of Scope)

MVP에서 구현하지 않는 항목:

- Claude AI 이력서 클리닉 (Phase 2)
- 결제 / 구독 플랜 (Phase 4)
- 모바일 앱 (웹 반응형으로 대체)
- 이력서 공유 SNS 연동 (링크 복사로 대체)
- 다국어 지원
- ATS(지원자 추적 시스템) 직접 연동

## 7. 성공 지표 (KPI)

| 지표 | MVP 목표 | 측정 방법 |
|------|---------|----------|
| 가입자 수 | 1,000명 | users 테이블 count |
| PDF 생성률 | 90% (가입자 대비) | resume_versions.pdf_s3_key not null 비율 |
| 피드백 수신율 | 40% (이력서 발행 대비) | feedbacks count / resumes published count |
| 웹 이력서 평균 체류시간 | 2분 이상 | view_sessions.total_duration_sec 평균 |
| 재방문율 | 60% (가입자 기준) | 월 2회 이상 로그인 유저 비율 |
